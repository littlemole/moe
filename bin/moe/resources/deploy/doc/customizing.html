Fx Keys

Keyboard Fx keys F1..F12 are supported. Each simply invokes a corresponding script in the user conf js folder, named F1..F12.js.
That's it.

The default is:

  • F1.js : Show Help
  • F2.js : New File
  • F3.js : Save All
  • F4.js : Save File
  • F5.js : Eval Script Code
  • F6.js : Insert Color
  • F7.js : Toggle Active Doc EOF type (UNIX/DOS)
  • F8.js : Show/Hide Dirview
  • F9.js : Show HTML Form
  • F10.js : Print
  • F11.js : Document Properties
  • F12.js : Moe Preferences

Toolbars

Moe supports more toolbars than displayed by default. You can change individual Toolbar Display under Menue -> View -> Toolbars. Moe comes with 5 grouped toolbars, and one User toolbar with all avail commands.

Note the option to unfreeze Toolbars with Menue -> View -> Toolbars -> freeze; this allows you to configure each toolbar with a right or doubleclick as well as to move them around (within some limits).

Menu & Toolbar extensions

The Tool Menue has four user configurable extension slots; for each a corresponding toolbar button is (optionally) displayable.

  • Batches
  • Scripts
  • Forms
  • Shortcuts

Shortcuts are the simplest; these are just macro text insertions into your current text document. Batches are Command-Line onliners to be executed in a (hidden) shell - anything you can run as a commandline can be put here, including .bat files.

Scripts allow you to run an ActiveScript compatible script file from withins moe embedded scripting host. On an usual Windows machine this would be vbscript and javascript assured; in case you have ActiveState Perl for windows installed, this will nicely do as well. These Scripts will have full access to moes object model as described under scripting in the moe help section.

Forms allow you to design a custom user interface (Form) using standard HTML. Event handling is possible by usual javascript (or vbscript, if you like); again with moes object model avail to the script handlers.

All four extensions can be configured via Menu -> Settings -> Preferences on tab User Settings. Alternatively export settings on the same tab into a clear text file.

Keybord Shortcuts

Keyboard Shortcuts are hardcoded and actually NOT customizable at this point. The shortcuts are:

  • CTRL-F: Find in files
  • CTRL-R: Replace in files
  • CTRL-N: New file
  • CTRL-O: Open file
  • CTRL-S: Save file
  • CTRL-D: Toggle Dirview
  • CTRL-W: Close
  • CTRL-SHIFT-W: Close All
  • ALT-S : Save file As
  • ALT-? : Help about

Moe Files overview

User folder

- {user}/AppData/moe/moe.xmo contains user Menu & Toolbar extensions (see above) in binary form (that is, Microsoft structured storage, see MSDN)

- {user}/AppData/moe/ui.xmo contains the current toolbar customization in binary serialized form (again, structured storage, see MSDN). This technique is a windows builtin, see MSDN.

It is possible to make copies of these files and just play them back (with all moe instances closed) as backup.

- fx folder Contains a small javascript which will be executed by moe when the corresponding keyboard fx key is pressed. These settings preceed the ones in the app folder which is thought as backup (and used by default if a user defined is missing).

Programm folder
moe.exe executable and local COM server.imageview and html/html-form view objects.
shellCtrls.dll treeview,dirview,hexview,colorpicker (com-dll)
scintillAx.dll editor component, Scintilla based (com-dll)
comTypes.dll COM typelib to XML exporter COM-Object. used in moe help HTML pages
io.dll IO abstractions to help in scripting. not used by moe internally.
pipedprocess.dll another scripting helper (initially written for tidy extension). not used by moe internally.
setting.dll generic user setting COM object. implements moe's Menu & Toolbar extensions and its GUI.
